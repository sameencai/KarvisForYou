# 模型配置指南

> Karvis 支持多种大模型平台。本文说明如何配置不同的模型。

---

## 模型配置位置

在 `.env` 文件中修改以下三个字段：

```bash
DEEPSEEK_API_KEY=你的API密钥
DEEPSEEK_BASE_URL=API地址
DEEPSEEK_MODEL=模型名称
```

> 虽然字段名叫 `DEEPSEEK_*`，但实际上支持任何兼容 OpenAI API 格式的平台。

---

## 平台配置速查

### DeepSeek 官方（推荐）

```bash
DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxxxx
DEEPSEEK_BASE_URL=https://api.deepseek.com/v1
DEEPSEEK_MODEL=deepseek-chat
```

**获取 API Key**：
1. 打开 [DeepSeek 平台](https://platform.deepseek.com/)
2. 注册账号 → API Keys → 创建
3. 充值 10 元

**可用模型**：`deepseek-chat`（自动指向最新 V3.2）、`deepseek-reasoner`

**推荐理由**：`deepseek-chat` 始终指向最新版模型，无需手动切换。

---

### 腾讯云知识引擎 lkeap

```bash
DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxxxx
DEEPSEEK_BASE_URL=https://api.lkeap.cloud.tencent.com/v1
DEEPSEEK_MODEL=deepseek-v3.2
```

**获取 API Key**：
1. 打开 [lkeap 控制台](https://console.cloud.tencent.com/lkeap)
2. 登录腾讯云账号
3. 开通服务 → API Key 管理 → 创建密钥
4. 充值 10 元即可使用

**可用模型**：`deepseek-v3.2`、`deepseek-r1-0528`

**优点**：国内网络稳定，无需翻墙，延迟低。

> ⚠️ 旧模型名 `deepseek-v3` 已下线，请使用 `deepseek-v3.2`。

> 参考文档：https://cloud.tencent.com/document/product/1772/115969

---

### 其他兼容 OpenAI API 的平台

只要平台提供兼容 OpenAI 的 API 接口，都可以接入。

```bash
DEEPSEEK_API_KEY=平台提供的API密钥
DEEPSEEK_BASE_URL=平台的API地址（通常以/v1结尾）
DEEPSEEK_MODEL=平台支持的模型名
```

**示例：硅基流动 (SiliconFlow)**
```bash
DEEPSEEK_API_KEY=sk-xxx
DEEPSEEK_BASE_URL=https://api.siliconflow.cn/v1
DEEPSEEK_MODEL=deepseek-ai/DeepSeek-V3
```

## 常见错误

### "模型不存在" / "Model not found"

模型名写错了。不同平台的模型名不一样：

| 平台 | 正确模型名 | 常见错误 |
|------|-----------|---------|
| DeepSeek 官方 | `deepseek-chat` | 写成了 `deepseek-v3` |
| 腾讯云 lkeap | `deepseek-v3.2` | 写成了 `deepseek-chat` 或已下线的 `deepseek-v3` |

### "API Key 无效" / "Unauthorized"

1. 检查 `DEEPSEEK_API_KEY` 有没有拼写错误
2. 检查 Key 是否已过期或被禁用
3. 检查 `DEEPSEEK_BASE_URL` 和 Key 是否来自同一个平台（lkeap 的 Key 不能用在 DeepSeek 官方，反之亦然）

### "余额不足"

去对应平台充值。Karvis 日常使用 token 消耗不多，10 元可以用很久。

## Qwen 模型（可选，省钱）

Karvis 支持配置第二个模型（Qwen），用于轻量任务（陪伴关怀、简单生成），减少 DeepSeek 的调用量。

```bash
QWEN_API_KEY=sk-xxxxxxxxxx
QWEN_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
QWEN_MODEL=qwen-plus-latest
QWEN_VL_MODEL=qwen-vl-max
```

**获取**：注册 [阿里云百炼](https://bailian.console.aliyun.com/)，创建 API Key。

不配置 Qwen 也完全可以，所有任务会走 DeepSeek。

---

## 视觉模型（图片理解）

发送图片给 Karvis 时，会使用视觉模型识别图片内容。默认使用 Qwen VL：

```bash
QWEN_VL_MODEL=qwen-vl-max
```

不配置 Qwen 时，图片会跳过视觉理解环节。
